Переход на линукс

2020-11-21

Работая с расбери пай, и сервером на digital ocean пришлось постепенно вникать в систему линукс, и узнавать основные команды. Спустя три года появилась мысль, а почему бы с основного компа не перейти на линукс. С опытом работы в линуксе ростет уверенность, что операционка достойная.

Был составлен план миграции

  1. Подготовить список установленных и используемых программ на windows.
  2. Проверка наличия официальных версий и поиск аналогов программ.
  3. Какия плюсы я получу от всего этого.

Чтобы переход был безболезненным и плавным можно запускать линукс на виртуальной машине с помощью VirtualBox, после изучения новой среды, поставить линукс основной системой. Также можно сделать из текущей ОС виндой виртуальную машину, котору можно будет впоследствие запустить на линукс, если очень будет нужно.

Делается это так, как описано здесь
https://community.spiceworks.com/how_to/148559-windows-10-physical-to-virtualbox
Программой Disk2vhd создается образ, причем его можно делать из загруженной ОС.
https://docs.microsoft.com/en-us/sysinternals/downloads/disk2vhd

Затем образ конвертируется в образ VirtualBox, затем при создании виртуалки не забыть установить флаг в настройках Enable EFI (special OSes only).
C:\Program Files\Oracle\VirtualBox .... VBoxManage clonehd <filename>.vhdx <filename>.vdi --format VDI
И больше действий от меня не потребовалось.

Список программ

Для чего мы используем операционную систему, если копировать файлы, то это можно сделать одинакого просто в любой из них, плюсом линукса будет бесплатность, и большая поддержа форматов дисков.

Кроме копирования файлов, еще простые задачи как посмотреть видео, открыть браузер походить по сайтам, послушать музыку, отредактировать документ, все это тоже можно сделать в любой системе, и предпочтительней та которая бесплатная, к примеру убунту. Выходит, что остаться на винде можно в том случае если нету аналога програм под линукс, или куплена программа только под винду, но есть возможность попытаться запустить ее в среде линукса, это wine, mono, virtual box (об этом позже).

Аналоги RDP

remmina

Лучше создать запись, с нее можно быстро подключиться по указанным ранее настройкам.

Пришлось немного воевать с тем, чтобы изображение по РДП не было смазанным. Ситуация такая, у меня два монитора, с разным разрешением, один моник с высоким разрешением, и на нем раньше винда сама выставляла масштабирование, и при подключении по РДП винды, настройки масштабирования применялись сразу к удаленному компу, сейчас это не работает так хорошо, но скорее есть решение лучше. Для того, чтобы сейчас экран не замыливался, масштабирование убунты системное, я убрал, но при подключении по РПД к компу винды, указываю, чтобы оно было 120%, иначе очень мелко. Минус в том, что теперь в системе убунты не будет масштабирования, но можно в приложения увеличить шрифт. Минусом ремины является возможность подключения только с одного монитора, для двух есть xfreerdp.

xfreerdp

Можно подключаться с двумя мониторами.
Масштабирование указывается общее для двух моников сразу, хотелось бы для каждого монитора отдельно, может такое и есть.
sudo apt install freerdp2-x11
xfreerdp /multimon /u:USER_NAME '/p:PASSWORD' /v:USER_NAME /scale:100 /scale-desktop:120

Команды

Получить все порты usb and com

ls /dev/ttyUSB* /dev/ttyUSB0 /dev/ttyUSB1

показать все диски и отмонтированые тоже

sudo fdisk -l

подключить нтфс диск на запись

sudo ntfsfix /dev/sdb1
sudo mkdir /mnt/media0
sudo mount /dev/sdc1 /mnt/media0

уменьшать размер образа РП
монтировать исо образ как папку
монтировать исо образ как физ. устройство и подключаться партионом прогой

file:///home/vadim/MY/PC_copy/Documents/!Linux/How%20to%20Backup%20and%20Restore%20Your%20Raspberry%20Pi%20Image%20-%20Agung%20Pratama%20-%20Blog.mhtml
losetup -d /dev/loop0

найти процес по окну

xprop | grep WM_CLASS WM_CLASS(STRING) = “gedit”, “Gedit”

Display the Path of Your Current Directory

$ pwd /home/scott/music/new

##### По умолчанию whereis выполняет поиск файлов команды в жестко заданных путях и каталогах, перечисленных в переменных среды. Используйте опцию -l, чтобы найти каталоги, где команда whereis ищет.

whereis bash whereis -l

scp - копирование файлов через ssh

https://wiki.enchtex.info/tools/console/scp

Скопировать локальный файл на сервер: scp file.gz root@server.my:/home/dir

Скопировать всё содержимое папки на сервере (рекурсивно) в локальную папку (с подробным выводом): scp -r root@server.my:/home/dir/ /home/local/my/
